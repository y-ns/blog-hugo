Hugo + GitHub Pagesでブログをはじめる

2022-03-03
tech
Hugo , GitHub Pages , Blog

Hugo + GitHub Pagesでブログをはじめる。

  • 静的サイトジェネレーター(SSG)の「Hugo」と 静的サイトのホスティングサービス「GitHub Pages」を使って簡単にブログをつくる。

つくってみる。

  • 作業時間: 1時間
  • 目標: 「Hugoで作成したサイトをGitHub Pagesで公開」
  • 環境:
    • macOS Big Sur
    • hugo v0.82.0+extended darwin/amd64
  • 参考:
    • Hugo
    • GitHub Pages

1. Hugoをインストール

    % brew install hugo

2. サイトを作成

    % hugo new site サイトの名前

3. テーマを選ぶ

  • Hugo Themes
  • 選んだテーマページに記載されているインストール方法に従う。
  • 当ブログのThemeは Cactusです🙏

4. 起動

    % hugo server

5. Gitリポジトリと紐付け

    % git init
    % git add .
    % git commit -m "first commit"
    % git remote add git rm --cached --origin リポジトリのURL
    % git push -u origin main

6. GitHub Pagesで公開

  • config.tomlに下記を追加
    • GitHub Pagesで公開する場合、docs以下に公開用のファイルを設置するルール。
    • Hugoの公開ファイルはpublic以下に生成されるが、この記述でdocs以下になる。
   publishDir = "docs"
  • 公開用のファイル生成
    % hugo
  • 生成されたファイルをpush
  • GitHub画面 > Settings > Pages を選択
    • Source: main を選択
    • Select folder: /docs
    • Save
  • 公開完了!

7. 独自ドメインの適応

  • GitHub Pagesと独自ドメイン

トラブルシューティング

ビルドエラー1

  • 内容: 
    Error: fatal: No url found for submodule path 'themes/cactus' in .gitmodules
Error: The process '/usr/bin/git' failed with exit code 128
  • 原因:
    • .gitmodulesファイルに’themse/cactus’の記述がなかった。そもそも.gitmodulesファイルが存在していなかった。
  • 対応: 
    % git submodule add https://github.com/monkeyWzr/hugo-theme-cactus.git themes/cactus --depth=1
'themes/cactus' already exists in the index

% git submodule update --init --recursive
fatal: No url found for submodule path 'themes/cactus' in .gitmodules

% git rm --cached themes/cactus
rm 'themes/cactus'

% git submodule add https://github.com/monkeyWzr/hugo-theme-cactus.git themes/cactus --depth=1
Adding existing repo at 'themes/cactus' to the index

% git submodule update --init --recursive

% git ls-files --stage | grep 160000
160000 a27a14348040b919818cbc862e3667c05f90b077 0	themes/cactus

% git status                        
On branch main
Your branch is up to date with 'origin/main'.

Changes to be committed:
(use "git restore --staged <file>..." to unstage)
    new file:   .gitmodules
    • .gitmodulesにテーマの記述が追加されていることを確認しpush
    [submodule "themes/cactus"]
path = themes/cactus
url = https://github.com/monkeyWzr/hugo-theme-cactus.git

ビルドエラー2

  • 内容: 
    Conversion error: Jekyll::Converters::Scss encountered an error while converting 'assets/css/style.scss':
No such file or directory @ dir_chdir - /github/workspace/docs
  • 原因:
    • 公開用ファイルを生成していなかった。
  • 対応: 
    % hugo
    • 公開用ファイルが生成されていることを確認しpushする。

独自ドメインを適応するとThemeが適用されない

  • 内容:
    • 独自ドメインを適応するとThemeが適用されない。
  • 原因:
    • config.tomlのbaseURLが<user>.github.ioのままだった。
  • 対応:
    • config.tomlのbaseURLを独自ドメインのURLに変更しpushする。

メモ。

ポートフォリオの公開にもよさそう。
